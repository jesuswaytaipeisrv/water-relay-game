# 使用說明

## 遊戲規則

1. 玩家只輸入名字加入等待名單，不自行選隊。
2. 主持人按「自動分隊」，系統會依加入順序平均分配到晨露隊、河浪隊與嫩芽隊，隊員人數不限。
3. 每次點擊會增加隊伍的一單位水量；所有隊員以 emoji 群眾顯示在左側取水起點，等待輪流接力。
4. 提滿主持人設定的打水次數，就完成一桶水。系統會讓一位隊員（🏃🪣）從賽道左側取水起點跑到右側終點灌溉（💦），倒水後折返；下一位在灌溉完成時同步從起點出發。
5. 終點五位小人（🧒）會同時依累積澆水比例平滑長大；每人達到主持人設定的成長目標後會長成大人（🧑✨）。
6. 最先讓五位小人全部長大的隊伍獲勝（🏆）。

主持人從左側控制面板操作分隊、設定、QR Code 與開始按鈕，並在右側查看三隊同場賽場；每隊一條橫向賽道，手機上會自動垂直堆疊、不需橫向捲動。玩家加入後的手機頁只保留自己的隊伍、點擊次數、隊伍成長進度條與打水按鈕，不會顯示比賽賽場。

## 主持流程

1. 開啟公開主持網址 [https://jesuswaytaipeisrv.github.io/water-relay-game/?view=host](https://jesuswaytaipeisrv.github.io/water-relay-game/?view=host)；系統會自動產生新的六碼房間，並將房間碼寫入網址。
2. 依活動需求調整「提滿一桶」的打水次數、「每人長大」所需淋水次數與倒數秒數。
3. 在主持台的「掃描加入」區塊顯示 QR Code，讓參賽者掃描後直接進入該新房間；也可按「複製連線網址」傳送給玩家。
4. 參賽者全部加入後，按「自動分隊」，確認所有人已分配到三隊；人數不受每隊五人限制。
5. 按「開始倒數」。
6. 比賽結束後，按「下一輪」建立乾淨的新回合。

若需回到既有場次，可直接使用 `?view=host&room=房間碼`。新建主持場次時，不需要手動輸入房間碼。


瀏覽 `http://127.0.0.1:5175/?view=host`。主持頁會自動產生房間碼；若 Firebase 未成功連線，程式會退回同一瀏覽器與相同網域分頁可用的示範模式，不能作為多手機活動測試。

主持台雖會顯示 QR Code，但 `127.0.0.1` 是目前電腦本身，手機掃描後無法連到你的電腦。多手機活動請使用部署後的 HTTPS 網址，並完成 Firebase 設定；頂端必須顯示「即時多人模式」。

## Firebase 多手機設定

目前公開版本已完成這些設定。若日後改用新的 Firebase 專案或重新建立資料庫，請依下列步驟重新設定：

1. 在 Firebase 專案新增 Web App，啟用 Realtime Database，並在 Authentication 的「登入方式」啟用「匿名」。
2. 以 Firebase Console 產生的 Web 設定填入 `firebase-config.js`。只填設定物件，不能把 Console 顯示的 npm 範例程式整段貼入。
3. 在 Realtime Database 的 Rules 頁貼上 `firebase-database.rules.json` 內容並發布。
4. 將本專案部署到 GitHub Pages、Zeabur 靜態網站或其他 HTTPS 網址。
5. 用部署後的網址開啟主持與玩家頁，確認頂端顯示「即時多人模式」。

## Firebase 規則與活動風險

規則範本需要匿名登入。程式會匿名登入後才連線資料庫，能避免未登入的直接讀寫，但不會阻止已加入者偽造更多點擊或重設資料。此限制對輕量現場遊戲可接受；若獎品或公平性要求高，應新增後端驗證或受控主持人 API。

Firebase 可能依讀寫量計費。小型短時活動通常可落在免費額度內，但沒有保證；先確認目前 Firebase 方案與預算警示。

QR Code 由隨專案發布的 MIT 開源 `qrcodejs` 函式庫在主持人的瀏覽器內產生，不會將玩家連線網址送交 QR 產生服務，也沒有額外付費或 CDN 依賴。

## 結束活動

在 Firebase Console 刪除 `water-splash-race/rooms/<房間碼>`，或直接刪除整個 `water-splash-race` 節點。不要將服務帳戶檔案或後端金鑰放進本專案。
